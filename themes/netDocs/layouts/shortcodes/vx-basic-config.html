After you have created the three VMs: leaf01, leaf02, and spine 01, power on the VMs, then perform the following basic configuration:

  • Change the hostname
  • Bring up swp1, swp2, and swp3
  • Check the connections

You can also check system information and upgrade Cumulus Linux using ONIE (an open source project (equivalent to PXE on servers) that enables the installation of network operating systems).

Change the Hostname

On each VM, run the following command, where <name> is leaf01, leaf02, or spine01, then exit the switch and log back in:

cumulus@cumulus:mgmt:~$ net add hostname <name>
cumulus@cumulus:mgmt:~$ net commit
cumulus@cumulus:mgmt:~$ exit

Bring up the Interfaces

On each VM, run the following command to bring up swp1, swp2, and swp3:

cumulus@leaf01:mgmt:~$ net add interface swp1,swp2,swp3
cumulus@leaf01:mgmt:~$ net commit

Check the Connections

On each VM, check the network connections and see which ports are neighbors of a given port:

cumulus@leaf01:mgmt:~$ net show lldp
LocalPort    Speed    Mode          RemoteHost     RemotePort  
-----------  -------  ------------  ------------   ------------
swp1         1G       Default       spine01        swp1
swp2         1G       Default       leaf02         swp2
swp3         1G       Default       leaf02         swp3

cumulus@leaf02:mgmt:~$ net show lldp
LocalPort    Speed    Mode          RemoteHost     RemotePort  
-----------  -------  ------------  ------------   ------------
swp1         1G       Default       spine01        swp2
swp2         1G       Default       leaf01         swp2
swp3         1G       Default       leaf01         swp3

cumulus@spine01:mgmt:~$ net show lldp
LocalPort    Speed    Mode          RemoteHost     RemotePort  
-----------  -------  ------------  ------------   ------------
swp1         1G       Default       leaf01         swp1
swp2         1G       Default       leaf02         swp1

Show System Information

To check system information on your VMs, including the Cumulus Linux version, run the net show system command:

Upgrade Cumulus Linux

Cumulus VX images include the GRUB boot loader and Open Network Install Environment (ONIE) preinstalled. You can install Cumulus Linux on switch hardware using a binary image. You can test this process by installing a Cumulus VX binary image with ONIE in a virtual environment.

  1. After booting the VM, reboot into ONIE Rescue mode using one of two methods:

    • Select ONIE Rescue mode on next reboot and reboot the VM with the sudo onie-select -rf && sudo reboot command.
    • Reboot and during the first 5 seconds on the GRUB menu, change the boot image to ONIE, then select ONIE Rescue Mode using the GRUB menu.
  2. To install Cumulus VX, run the onie-nos-install <URL to cumulus-linux-vx-amd64.bin> command.
